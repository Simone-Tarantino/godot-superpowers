# Export Config

Configure Godot exports cleanly for distribution. Reference: [Exporting projects](https://docs.godotengine.org/en/stable/tutorials/export/exporting_projects.html).

## Prerequisites

1. **Export templates installed**: Editor → Manage Export Templates → Download (matches your editor version)
2. **`export_presets.cfg`** in `.gitignore` (it can contain keystore paths and signing secrets); commit `export_presets.cfg.example` if useful
3. **Icon assets**: Source `.png` 1024×1024 for the icon; Godot generates platform variants

## `export_presets.cfg` structure

This file is auto-generated by Godot but understandable. Each preset is a numbered section:

```ini
[preset.0]

name="Windows Desktop"
platform="Windows Desktop"
runnable=true
dedicated_server=false
custom_features=""
export_filter="all_resources"
include_filter=""
exclude_filter="docs/*, test/*, *.gdignore, *.md"
export_path="builds/windows/MyGame.exe"
encryption_include_filters=""
encryption_exclude_filters=""
encrypt_pck=false
encrypt_directory=false

[preset.0.options]

custom_template/debug=""
custom_template/release=""
binary_format/embed_pck=false
debug/export_console_wrapper=1
binary_format/architecture="x86_64"
codesign/enable=false
application/icon="res://icon.ico"
application/file_version="1.0.0.0"
application/product_version="1.0.0.0"
application/company_name="MyStudio"
application/product_name="MyGame"
application/file_description="MyGame"
application/copyright="© 2026 MyStudio"
ssh_remote_deploy/enabled=false
```

Recommended: configure presets through the editor UI (Project → Export). It writes the file. Then commit a sanitized template.

## Per-platform presets

### Windows Desktop

- **Architecture**: `x86_64`
- **Embed PCK**: `false` (separate `.pck` ships smaller patches; `true` for single-file)
- **Code signing**: `osslsigncode` or `signtool` (Windows SDK). For most indie releases, leave unsigned (users see a SmartScreen warning that goes away after enough downloads).
- **Icon**: `.ico` file via `application/icon`
- **Console wrapper**: `1` for debug builds, `0` for release (no console window)

### macOS Desktop

- **Architecture**: `universal` (x86_64 + arm64) — required for Apple Silicon
- **Bundle identifier**: reverse DNS, e.g. `com.mystudio.mygame`
- **Codesign**: `application/codesign/enable = true` and provide a Developer ID certificate; otherwise users see Gatekeeper warning
- **Notarization**: required for distribution outside Mac App Store; via `xcrun notarytool` or Godot's built-in
- **Icon**: `.icns` file

### Linux

- **Architecture**: `x86_64` (also `arm64` if targeting Pi / Steam Deck)
- **Distribution format**: AppImage (recommended for desktop), tarball, or Flatpak
- **Embed PCK**: `true` — single binary easier to distribute
- **Set executable**: chmod +x post-build

### Web (HTML5)

- **Custom HTML shell**: optional but useful for branding the loading screen
- **Threads**: requires CORS headers (`Cross-Origin-Embedder-Policy: require-corp`, `Cross-Origin-Opener-Policy: same-origin`); itch.io has a setting for this
- **Embed PCK**: `true` (single `.pck` request fewer)
- **VRAM compression**: `true` for smaller download
- **Limits**: 2GB max memory; large textures can crash mobile browsers

### Android

- **Java SDK + Android SDK** required; configure paths in Editor Settings → Export → Android
- **Keystore**: generate with `keytool -genkey -v -keystore my.keystore -alias mygame -keyalg RSA -validity 10000`. Store outside repo. Never commit.
- **Min SDK**: 24 (Godot 4.4 default); covers Android 7+
- **Target SDK**: latest (required by Play Store)
- **Architectures**: `arm64-v8a` (required for Play Store), optionally `armeabi-v7a` for older devices
- **Permissions**: declare only what's needed (Internet for online; nothing for offline single-player)

### iOS

- **Xcode** required (macOS-only)
- **Apple Developer account** with provisioning profile
- **Team ID + Bundle ID** in preset
- **App icons**: full set (use [appicon.co](https://appicon.co) or similar)
- **Architectures**: `arm64`
- **Use TestFlight** for beta distribution

## Project icon

Source: `icon.png` 1024×1024 with transparent background. Godot can convert to platform variants but `.ico` (Windows) and `.icns` (macOS) need manual generation:

```bash
# .ico (Windows) via ImageMagick
convert icon.png -define icon:auto-resize=256,128,96,64,48,32,16 icon.ico

# .icns (macOS)
mkdir icon.iconset
sips -z 16 16     icon.png --out icon.iconset/icon_16x16.png
sips -z 32 32     icon.png --out icon.iconset/icon_16x16@2x.png
sips -z 32 32     icon.png --out icon.iconset/icon_32x32.png
sips -z 64 64     icon.png --out icon.iconset/icon_32x32@2x.png
sips -z 128 128   icon.png --out icon.iconset/icon_128x128.png
sips -z 256 256   icon.png --out icon.iconset/icon_128x128@2x.png
sips -z 256 256   icon.png --out icon.iconset/icon_256x256.png
sips -z 512 512   icon.png --out icon.iconset/icon_256x256@2x.png
sips -z 512 512   icon.png --out icon.iconset/icon_512x512.png
sips -z 1024 1024 icon.png --out icon.iconset/icon_512x512@2x.png
iconutil -c icns icon.iconset
```

## CLI build commands

```bash
# Debug build (fast, includes debug symbols)
godot --headless --path . --export-debug "Windows Desktop" builds/windows/MyGame_debug.exe

# Release build (optimized)
godot --headless --path . --export-release "Windows Desktop" builds/windows/MyGame.exe

# All presets (run separately):
godot --headless --path . --export-release "Windows Desktop" builds/windows/MyGame.exe
godot --headless --path . --export-release "macOS"           builds/macos/MyGame.dmg
godot --headless --path . --export-release "Linux/X11"       builds/linux/MyGame.x86_64
godot --headless --path . --export-release "Web"             builds/web/index.html
godot --headless --path . --export-release "Android"         builds/android/MyGame.apk
```

`--export-debug` if the platform supports it, `--export-release` otherwise. The preset name must match exactly what's in `export_presets.cfg`.

## Pre-flight checklist

Before any release export:

- [ ] Version bumped in `project.godot` (`config/version="1.2.3"`)
- [ ] Icon set on every platform preset
- [ ] Bundle/app ID consistent across platforms
- [ ] `exclude_filter` strips `test/`, `docs/`, `tools/`, `.gdignore`
- [ ] `print()` / `print_debug()` calls behind a debug flag
- [ ] Logging level set to WARN or ERROR for release
- [ ] Autoloads load without errors when scene starts cold
- [ ] All addons under `addons/` are licensed for distribution (some are dev-only)
- [ ] `export_presets.cfg` not committed (or sanitized version only)
- [ ] No hardcoded paths from your machine in scripts
- [ ] `user://` paths used for save data, not absolute paths
- [ ] `display/window/size/viewport_*` matches design resolution
- [ ] No shaders that crash on the lowest-spec target hardware
- [ ] Test build runs on a clean machine (not just dev box)

## CI build (GitHub Actions)

`.github/workflows/build.yml`:

```yaml
name: Build
on:
  push:
    tags: ['v*']

jobs:
  build:
    strategy:
      matrix:
        include:
          - { os: ubuntu-latest,  preset: "Linux/X11",       output: builds/linux/MyGame.x86_64 }
          - { os: windows-latest, preset: "Windows Desktop", output: builds/windows/MyGame.exe }
          - { os: macos-latest,   preset: "macOS",           output: builds/macos/MyGame.zip }
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: chickensoft-games/setup-godot@v2
        with:
          version: 4.4.1
          use-dotnet: false
      - name: Import resources
        run: godot --headless --import || true
      - name: Export
        run: |
          mkdir -p $(dirname "${{ matrix.output }}")
          godot --headless --path . --export-release "${{ matrix.preset }}" "${{ matrix.output }}"
        shell: bash
      - uses: actions/upload-artifact@v4
        with:
          name: ${{ matrix.os }}
          path: ${{ matrix.output }}
```

Tag `v1.2.3` → builds for all three desktop platforms upload as artifacts.

For mobile/web, use platform-specific runners (macOS for iOS, Linux for Android signing).

## Common gotchas

- **Build hangs on first export**: usually stuck importing assets. Run `godot --headless --import` first to warm the cache.
- **Web build crashes mobile browsers**: too much VRAM. Reduce texture sizes, disable shadows.
- **macOS unsigned build "is damaged"**: `xattr -cr /Applications/MyGame.app` removes the quarantine bit (for testers; ship signed builds for users).
- **Android signing key lost = can't update Play Store listing.** Back up the keystore.
- **Translation files**: list `*.translation` in `include_filter` if you use them.
- **`embed_pck=false`** + missing `.pck` next to executable = silent failure. Either embed or distribute both.
